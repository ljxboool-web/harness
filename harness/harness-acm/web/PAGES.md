# CF-Profiler Web 页面说明

CF-Profiler Web 是一个单页应用，左侧导航切换不同功能区。普通使用主要看“选手画像”，调试和验证时再看后面几个页面。

## 1. 选手画像

这是主页面，用来分析一个 Codeforces 选手的整体能力。

输入 handle 并点击“分析”后，页面会展示：

- 选手基础信息：rating、peak、比赛次数、AC 率
- 8 维算法技能：DP、图论、数学、贪心、数据结构、字符串、搜索、几何
- 5 维特征：稳定性、速度、抗压、攻坚、活跃度
- 难度分布：选手主要 AC 哪些难度段的题
- Verdict 分布：AC、WA、TLE、MLE 等结果比例
- Rating 轨迹：历史 rating 变化
- 近 180 天提交热力图
- 代码风格筛查：粘贴或上传源码后检查结构、命名、复杂度和可维护性风险
- 教练评语：调用模型生成“强项 / 弱项 / 建议”
- AI 训练题单：根据 rating 和低分技能维度推荐未尝试过的 Codeforces 题目

这个页面面向最终用户，回答“这个选手强在哪里、弱在哪里、下一步该练什么”。

代码风格筛查只分析用户提供的源码文本。Codeforces 官方 API 不返回提交源码，因此 Web 端不会自动抓取选手历史代码。

## 2. 错误原理

这个页面用于解释选手错误主要发生在哪里，以及这些错误类型通常意味着什么。

它基于 `/api/analyze` 已返回的聚合统计做启发式判断：

- AC 率和失败提交数
- WA / TLE / MLE / RTE / CE 等 verdict 分布
- 各难度段 solved / attempted
- 各标签 solved / attempted
- rated 与 practice 的 AC 率差异
- 总提交数与去重尝试题数的比例

页面会给出主要错误类型、风险等级、重复提交倍数、错误类型解释、高风险难度段、高风险标签和场景信号。

这个页面面向训练复盘，回答“错误更像是思路问题、复杂度问题、边界问题，还是比赛场景下的流程问题”。

## 3. Judge Loop

这个页面用于查看 AI 评语的质量审查过程。

生成评语后，系统会让 judge 检查评语是否合格，例如：

- 是否包含“强项 / 弱项 / 建议”三段
- 建议是否具体
- 是否引用了不存在的数据
- 数字是否和真实分析结果一致

页面会展示尝试轮次、最终分数、各 judge 的分数和 reason。

当前项目可以通过 `DASHSCOPE_JUDGE_MODE=fast` 使用快速模式，只跑单路 judge；默认完整模式会跑多个 judge 做交叉审查。

## 4. Baseline 对比

这个页面用于比较当前分析结果和历史基线。

适合在修改评分公式、tag 分类、聚合逻辑或 prompt 后使用。它可以帮助判断：

- 当前分数和旧分数差多少
- 哪些技能维度发生明显变化
- 哪些个性特征出现 drift
- 变化是否超过设定阈值

这个页面主要面向开发和回归验证，用来防止系统改动导致评分异常漂移。

## 5. Harness 健康度

这个页面用于查看系统运行状态和日志摘要。

当出现抓取失败、生成评语慢、模型调用异常或页面数据不对时，可以查看这里。它通常包括：

- API 调用情况
- cache hit / miss
- fetch 错误
- judge 运行记录
- metrics 统计
- 最近日志摘要

这个页面主要用于排障和运维。

## 推荐使用顺序

普通使用：

```text
选手画像 -> 生成评语
```

开发或调试：

```text
选手画像 -> 错误原理 -> Judge Loop -> Baseline 对比 -> Harness 健康度
```
